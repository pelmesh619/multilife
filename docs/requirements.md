# Функциональные требования

Проект представляет собой многопользовательскую среду, в которой существует двумерный клеточный мир, развивающийся по заданным правилам.

Игроки могут:

* подключаться к серверу
* наблюдать за симуляцией
* влиять на него, изменяя состояние отдельных клеток или областей

Пользователь может иметь роль:

* Игрок – основной пользователь, имеющий право просматривать мир и изменять клетки в рамках установленных ограничений
* Администратор – создатель сервера. Администратор имеет права на управление сервером: запуск/остановка симуляции, изменение правил, сброс мира, управление игроками

---

## Требования к серверу

* Сервер хранит состояние клеточного мира в виде двумерной сетки фиксированного размера

    Каждая клетка принадлежит какому-либо игроку и может находиться в одном из двух состояний: "живая" или "мертвая"

    Сервер циклически вычисляет следующее поколение мира по правилам, которые могут быть предопределены или настраиваться администратором

    При вычислении следующего поколения сервер не учитывает изменения, вносимые игроками во время этого же тика. Вместо этого изменения применяются к текущему состоянию и будут учтены в следующем поколении

* Сервер ожидает подключения клиентов по протоколу TCP на заданном порту. При подключении клиента сервер отправляет ему текущее состояние мира и игровые правила (информацию об ограничениях и так далее)
* Сервер поддерживает список подключенных клиентов с их идентификаторами и правами

* Клиенты могут отправлять серверу запросы на изменение состояния одной или нескольких клеток (например, координаты и новое состояние)

    Сервер проверяет допустимость действия в соответствии с правилами:

    * Клетка должна находиться в пределах сетки.
    * Игрок не превысил лимит на количество изменений за последний интервал времени

    Правильные изменения применяются к текущему состоянию мира немедленно до следующего тика эволюции

* Сервер информирует всех подключенных клиентов об изменениях в мире:
    * При наступлении нового тика рассылается информация об измененных клетках
    * При изменениях, инициированных игроками, рассылаются соответствующие обновления 

## Требования к клиенту

* Главное окно клиента содержит область просмотра клеточного мира в виде сетки, где каждая клетка окрашена в соответствии с ее состоянием

    Клиент имеет возможность масштабировать поле с помощью колесика мыши и перемещаться по карте с помощью перетискивания мышью

    Клиент должен отображать текущий номера поколения, количества живых клеток и других метрик

* Игрок может выбрать пустую клетку и захватить ее, потратив определенное число ресурса. Ресурс генерируется каждый тик пропорционально от числа захваченных клеток. Отправка изменений на сервер происходит либо немедленно при каждом действии, либо по кнопке "Применить"

* Клиент поддерживает постоянное соединение с сервером и асинхронно принимает обновления состояния. Полученные изменения отображаются на экране. При разрыве соединения клиент пытается переподключиться

* При запуске клиент показывает диалог ввода адреса сервера и порта. Можно сохранять настройки последнего успешного подключения. После подключения отображается статус соединения.

* Клиент отображает:
    * Список подключенных игроков с указанием их имен или идентификаторов
    * Кнопку отключения от сервера

## Требования к игровому процессу

* По умолчанию используются классические правила игры «Жизнь»:
    * Клетка становится живой, если у нее ровно 3 живых соседа
    * Клетка остается живой, если у нее 2 или 3 живых соседа
    * В остальных случаях клетка умирает или остается мертвой

* Правила могут быть изменены администратором. Сервер применяет правила одновременно ко всем клеткам в конце каждого тика.

* Для предотвращения злоупотреблений сервер устанавливает лимиты:
    * Максимальное количество изменяемых клеток в секунду на одного игрока.
    * Максимальный размер области за одно действие

    При превышении лимита изменение отклоняется, клиент получает уведомление

* Интервал между тиками эволюции задается администратором (по умолчанию 1 секунда). При необходимости эволюция может быть приостановлена (пауза)

* Все клиенты видят одно и то же состояние мира. Действия игроков применяются немедленно и рассылаются всем
